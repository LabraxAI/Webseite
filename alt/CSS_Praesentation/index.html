Geschichte von CSS:
CSS (kurz für Cascading Style Sheets) ist zusammen mit HTML und JavaScript eine der Kernsprachen des Internets. Sie wurde ursprünglich von Hakon Wium Lie und Bert Bos entwickelt. Doch das World Wide Web Consortium (W3C) zeigte sein intesse an der noch nicht einmal vollendeten Software und nachdem 1996 die erste CSS Version veröfentlicht worden ist, übernahm W3C immer mehr Teile des Projektes bis sie schließlich nahezu das ganze Projekt unter ihre Fittiche nahmen. CSS wird auch heute noch unter ihrer Aufsicht veröffentlicht und verbessert.

Doch wofür braucht man CSS?
CSS ist ein Sprache, welche primär für die nähere Formatierung und Ausgestaltung von Webseiten gedacht ist, da Sprachen wie HTML nur wenige solche Formatierungsmöglichkeiten bieten. Es arbeitet mit Sprachen wie HTML zusammen, braucht jedoch auch eine seperate CSS-Datei, um ihr volles Potential zu entfalten, und damit kommen wir auch gleich zum nächsten Punkt: der da heißt

Wie funktioniert CSS?
Um CSS einsetzen zu können braucht ihr neben der HTML-Datei, in welcher sich eure Seite befindet, auch eine CSS-Datei. Zwar können einige CSS Befehle auch ohne zusätzliche CSS-Datei eingesetzt werden. Doch kann so nicht das volle Potential der Sprache ausgenutzt werden. CSS arbeitet mit zwei primären Bezeichnungen:
1.Selektoren, mit welchen man den zu bearbeitenden Teil der Webseite auswählen kann
und 2. die Attribute, mit denen der ausgewälte Teil dann verändert werden kann. Atribute können z.B. Größe, Breite, Farbe usw. bestimmen!

Versionen von CSS:
CSS wurde bisher in insgesamt 3(genaugenommen 4) Versionen veröffentlicht.
CSS 1(oder auch CSS Level 1) wurde 1996 mit dem Ziel, HTML-Seiten besser formatieren zu können, veröffentlicht.
CSS 2 wurde bereits 1998 veröfentlicht, war aber mit nahezu keinem Browser kompatibel, weswegen 2010 eine verbesserte version von CSS 2 namens CSS 2.1 veröffentlicht wurde. Diese Version war nun auch mit einem Großteil der Browser kompatibel.
Die Arbeiten an CSS 3 begannen bereits im Jahr 2000, doch, als man die Probleme von CSS 2 erkannte, beschlos man der Behebung eben dieser Probleme zunächst höchste Priorität zu geben. So kam es, dass CSS 3 immer noch nicht veröffentlicht wurde. Eine vierte Version von CSS ist bisher nicht geplant. Weiter

Das Testfeld

Nun fahren wir mit den Selektoren fort. Man hat mehrere Möglichkeiten, in CSS Elemente der HTML-Datei auszuwählen. Die wohl einfachste ist, einfach den Namen des HTML-Tags anzugeben. Das sähe z.B. bei einem Link so aus: a {Attribute}. Diese Methode ist sehr simpel, hat jedoch den Nachteil, dass man nicht zwischen unterschiedlichen Elementen, die den selben Tag haben, unterscheiden kann. Für solche Fälle sind IDs und Klassen gedacht. Der Unterschied zwischen den beiden ist, dass eine ID nur für ein bestimmtes Element gelten kann, während man eine Klasse beliebig vielen Elementen zuweisen kann. Einem Bild könnte man z.B. so eine ID zuweisen: <img id="BeispielID" src="http://placehold.it/100x100">. Eine Klasse könnte man wiederum so zuweisen: <img class="BeispielKlasse" src="http://placehold.it/100x100">. Im CSS-Code könnte man diese Klasse dann so ansprechen .BeispielKlasse {Attribute}. Mit der ID ginge das dann so #BeispielID {Attribute}

Dies allein reicht aber nicht, um für alle Fälle gerüstet zu sein, z.B kann man damit nicht dafür sorgen, dass ein Element nur ausgewält wird, wenn man mit der Maus drüber fährt oder es geklickt wird. Für diese Fälle gibt es sogenannte Pseudoklassen. Dieses wendet man so an, dass man hinter den Teil des Selektors, der das Element auswählt, ein bis zwei Doppelpunkte(je nach Pseudoklasse unterschiedlich) setzt und dann den entsprechenden Befehl eingibt.
